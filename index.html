creatHexo

发表于 2019-08-11 | 分类于 提醒自己防止忘记 | 阅读次数:

#个人博客的使用记录
hexo+github搭建个人网站，以此来记录下来一些基本操作，防止自己因为忙忘了操作再去查询

##在本地进行预览命令
hexo server
在浏览器输入http://localhost:4000/进行预览

##对文章进行设置（归类，创建标签，创建分类）

###创建分类页面命令
hexo new page categories

####基本设置

##新建一个文章命令
hexo new a
a是文章的标题，也可以加双引号例如”a”。此时新建的是一个post page而不是草稿(draft page)

#新建一个草稿箱

`hexo new draft b`

会在source下看见一个新文件夹——_drafts，里面装了我们写好的草稿
我们可以利用以下命令进行预览草稿文章,而不进行发布

`hexo server --draft`

我们可以用下列命令进行发布草稿

`hexo publish b`

他会把_drafts里面的b草稿放到_posts里面

##新建“关于”、“了解我们”之类的页面
这个命令会在source文件夹里面创建出C文件夹，里面会有一个index.md文件

`hexo new page c`

这个可以用http://localhost:4000/c/进行访问(其不会出现在博客列表里)

##设置默认的生成方式
在_config.yml文件可以更改设置，比如下面的命令可以设置默认的为draft而不是post 

default_layout:draft

##发布本地文章到github上命令
hexo clean && hexo g && hexo d

hexo clean 清除本地缓存
hexo g generate的缩写,生成静态的html
hexo d deploy的缩写，部署到网站上

#背景介绍
由于项目要求实现一个视频聊天功能的软件，实现手机端到PC端，手机和手机端视频聊天，甚至包含屏幕分享功能，查阅了大量资料，鉴于对C++情有独钟，故开始了QT库的学习使用，利用其优良的跨平台特性。但是尝试了各种第三方库，似乎都不支持QT,无奈只能QT和web混合开发，但是奈何其兼容性并不好，于是乎选择自己造轮子，选择了强大的opencv库，巴拉巴拉开始安装，之前界面32位项目，但是安装opencv都是64位的，只能下载opencv源码自己编译成32位的，…… ……此处省略安装时反复跳坑填坑过程……，最后安装好了。
